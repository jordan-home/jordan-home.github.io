    Prompt Engineering 基础指南

    Jordan 2026/2/18 AIPrompt

    # 前言

    Prompt Engineering（提示工程）是与 AI 模型高效对话的艺术。本文介绍如何编写高质量的 Prompt。

"写关于编程的"

# ✅ 具体

    # 常见错误

    1. 过于笼统：模型无法准确理解需求
    2. 信息不足：缺少关键上下文
    3. 期望过高：单次 Prompt 完不成复杂任务
    4. 格式混乱：输出难以解析

    # 高级技巧

    # 1. ReAct（推理+行动）

    让模型先思考再行动：

    问题：北京明天天气如何？

思考：我需要先查询北京的天气API或搜索天气信息。
行动：搜索"北京天气 明天"
观察：搜索结果显示明天晴转多云，15-25度。
回答：北京明天晴转多云，气温15-25度。

    # 2. self-consistency（自一致性）

    让模型生成多个答案，取多数一致的结果：

    问题：100以内有多少个质数？

请生成3个不同的答案，然后分析哪个最准确。

    # 3. Tree of Thought（思维树）

    多路径思考：

    问题：如何优化首页加载速度？

分支1 - 资源优化：
- 压缩图片
- 代码分割
- CDN加速

分支2 - 缓存策略：
- 浏览器缓存
- Service Worker
- SSR

请比较两个方向的优缺点。

    # 常见错误

    1. 过于笼统：模型无法准确理解需求
    2. 信息不足：缺少关键上下文
    3. 期望过高：单次 Prompt 完不成复杂任务
    4. 格式混乱：输出难以解析
    5. 忽略限制：未说明输出长度、语言等限制

    # 最佳实践 Checklist

    □ 明确任务目标
□ 提供必要上下文
□ 指定输出格式
□ 说明长度限制
□ 包含示例（如需要）
□ 设定角色（如适用）
□ 检查是否可执行
