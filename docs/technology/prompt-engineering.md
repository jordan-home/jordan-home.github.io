## 前言

Prompt Engineering（提示工程）是与 AI 模型高效对话的艺术。本文介绍如何编写高质量的 Prompt。

## 核心原则

### 1. 明确具体

```python
# ❌ 模糊
"写关于编程的"

# ✅ 具体
"写一篇500字的文章，介绍JavaScript闭包的应用场景"
```

### 2. 提供上下文

```python
# ❌ 无上下文
"如何修复bug?"

# ✅ 有上下文
"作为Vue3开发者，遇到组件间数据不同步问题，provide/inject和mitt三种方案各有什么优缺点?"
```

### 3. 指定格式

```python
# ❌ 未指定格式
"列出学习资源"

# ✅ 指定格式
"用Markdown表格列出前端学习资源，包含：课程名称、平台、免费/付费、推荐理由"
```

## 常用技巧

### 1. Few-Shot Prompting

```python
# 示例学习
任务：将句子转换为JSON格式

示例：
输入：今天天气真好
输出：{"情绪": "positive", "主题": "天气"}

请转换：
输入：路上堵车迟到
输出：
```

### 2. Chain of Thought

```python
# 让模型展示推理过程
"请分步骤思考：如何从零开始学习机器学习？每一步说明目标和所需时间"
```

### 3. Role Prompting

```python
"你是一位有10年经验的前端架构师，精通React、Vue、性能优化。请评价这个代码的架构设计：\n\n[代码]"
```

### 4. 限制输出格式

```python
"用JSON格式返回，禁止其他文字：
{
  "name": "string",
  "age": "number", 
  "skills": ["string"]
}"
```

## 结构化 Prompt

```
# 角色
你是一位专业的技术博主

# 任务
写一篇关于React hooks最佳实践的文章

# 要求
1. 包含代码示例
2. 面向中级开发者
3. 包含常见错误和解决方案

# 格式
- 使用Markdown
- 代码块要有注释
- 结尾要有总结
```

## 实战案例

### 1. 代码审查

```
作为资深前端工程师，审查以下代码：
1. 性能问题
2. 安全问题  
3. 可维护性
4. 最佳实践

[代码]
```

### 2. 学习规划

```
我是一名有1年经验的前端开发者，想在6个月后找到React相关工作。
请制定学习计划，包含：
- 每周目标
- 每日学习时间安排
- 实战项目建议
- 面试准备建议
```

### 3. 内容生成

```
生成5个关于"前端性能优化"的博客标题，要求：
- 吸引点击
- 包含具体数字
- 面向初级开发者
```

## 常见错误

1. **过于笼统**：模型无法准确理解需求
2. **信息不足**：缺少关键上下文
3. **期望过高**：单次 Prompt 完不成复杂任务
4. **格式混乱**：输出难以解析

## 高级技巧

### 1. ReAct（推理+行动）

让模型先思考再行动：

```
问题：北京明天天气如何？

思考：我需要先查询北京的天气API或搜索天气信息。
行动：搜索"北京天气 明天"
观察：搜索结果显示明天晴转多云，15-25度。
回答：北京明天晴转多云，气温15-25度。
```

### 2. self-consistency（自一致性）

让模型生成多个答案，取多数一致的结果：

```
问题：100以内有多少个质数？

请生成3个不同的答案，然后分析哪个最准确。
```

### 3. Tree of Thought（思维树）

多路径思考：

```
问题：如何优化首页加载速度？

分支1 - 资源优化：
- 压缩图片
- 代码分割
- CDN加速

分支2 - 缓存策略：
- 浏览器缓存
- Service Worker
- SSR

请比较两个方向的优缺点。
```

### 4. XML/JSON 结构化输出

```
请用以下XML格式返回，不要有其他内容：
<article>
  <title>标题</title>
  <content>正文</content>
  <tags>
    <tag>标签1</tag>
    <tag>标签2</tag>
  </tags>
</article>
```

## 常见错误

1. **过于笼统**：模型无法准确理解需求
2. **信息不足**：缺少关键上下文
3. **期望过高**：单次 Prompt 完不成复杂任务
4. **格式混乱**：输出难以解析
5. **忽略限制**：未说明输出长度、语言等限制

## 最佳实践 Checklist

```
□ 明确任务目标
□ 提供必要上下文
□ 指定输出格式
□ 说明长度限制
□ 包含示例（如需要）
□ 设定角色（如适用）
□ 检查是否可执行
```

## 总结

- 明确具体，提供上下文
- 使用示例，指定格式
- 复杂任务分步骤
- 高级技巧：ReAct、self-consistency、ToT
- 持续迭代优化 Prompt
